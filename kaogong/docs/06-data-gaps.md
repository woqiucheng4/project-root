## 二、P0 数据详解（不补齐就不能商用）

### 2.1 专业目录分类表

**现状问题**：当前竞争评估模块用 `major.includes("法")` 这种字符串匹配判断专业，极其粗糙。用户输入"法学"和"法律事务"会得到相同结果，但"刑事科学技术"就匹配不上。

**需要的数据**：
```
国家公务员考试专业分类目录（教育部版本）
├── 法学类：法学、知识产权、监狱学、法律事务...
├── 计算机类：计算机科学与技术、软件工程、网络工程...
├── 中文类：汉语言文学、汉语言、中国语言文化...
├── 经济类：经济学、金融学、财政学...
├── 管理类：工商管理、行政管理、公共事业管理...
└── ...（约 13 个大类、100+ 小类）
```

**数据来源**：
- 国家公务员局官网公开的《专业分类指导目录》
- 各省人社厅补充发布的省级专业目录
- 格式：可整理为 JSON 配置文件

**影响范围**：
- 规则引擎的"专业匹配"过滤
- 竞争评估的"专业池大小"系数
- AI分析时对用户专业优势的研判

---

### 2.2 真实职位表数据库

**现状问题**：当前 `ruleEngine.js` 里硬编码了 5 个笼统的职位类型（两办/纪委、普通综合岗、乡镇基础岗、应届生定向、基层定向岗）。这不是真实岗位，只是分类标签。

**需要的数据**：
```
每年国考/省考职位表（Excel格式，公开数据）
├── 招录机关：XX省XX市XX区纪委
├── 职位名称：综合岗位
├── 职位代码：300110001
├── 招录人数：2
├── 学历要求：本科及以上
├── 专业要求：法学类
├── 政治面貌：中共党员
├── 基层经历：两年以上
├── 备注：限应届…
└── ...
```

**数据来源**：
- 国家公务员局每年10月发布的国考职位表（Excel）
- 各省人事考试网发布的省考职位表
- 公开免费，无版权问题

**建议方案**：
1. **MVP增强版**：手动下载 2024/2025 国考职位表 Excel，用脚本解析为 JSON 导入
2. **V2版本**：搭建爬虫定期抓取，或接入第三方数据源

## 三、P1 数据详解（直接影响分析质量）

### 3.1 历年报录比数据

**现状问题**：`competition.js` 用固定 60:1 作为默认基础报录比。实际上不同岗位差异巨大（三不限可达 3000:1，限条件岗可能只有 5:1）。

**需要的数据**：
```
近3年（2023-2025）国考/省考各岗位报名人数与录取人数
├── 职位代码：300110001
├── 计划录用：2人
├── 报名人数：856人
├── 过审人数：723人
├── 报录比：362:1
└── ...
```

**数据来源**：
- 国家公务员局公布的报名最终数据
- 各省考试院在报名期间公布的每日报名统计
- 部分数据可在公考培训机构网站（华图、粉笔等）找到整理版

**使用方式**：
- 替换竞争评估中的固定 `defaultBaseRatio = 60`
- 按岗位类型、地区、专业限制条件建立报录比预测模型

---

### 3.2 历年进面分数线

**现状问题**：当前没有任何分数参考数据。用户填了"行测70分"，系统无法判断这个分数在目标岗位中的竞争力。

**需要的数据**：
```
近3年各岗位的笔试入面最低分 / 录取最低分
├── 国考：副省级 行测+申论 合计 130-145 分
├── 国考：地市级 合计 120-135 分
├── 省考：各省差异较大
└── ...
```

**数据来源**：
- 国家公务员局公开的面试名单（可反推分数线）
- 各省公务员局公布的笔试成绩及入面分数线
- 培训机构整理好的分数线汇总

**使用方式**：
- 给用户的"上岸概率"评估提供真实基准线
- AI分析中给出"你的分数在XX岗位排名约 XX%"

---

### 3.3 地区行政区划库

**现状问题**：当前用 `region.includes("北")` 判断是否一线城市。用户输入"北海市"也会被误判为一线。

**需要的数据**：
```json
{
  "一线": ["北京市", "上海市", "广州市", "深圳市"],
  "新一线": ["杭州市", "成都市", "南京市", "武汉市", ...],
  "二线": ["大连市", "厦门市", "昆明市", ...],
  "三线及以下": [...],
  "偏远地区": ["西藏", "新疆南疆", "青海", ...]
}
```

**数据来源**：
- 民政部官网《中华人民共和国行政区划》
- 第一财经每年发布的《城市等级划分》
- 现成的 npm 包 `china-area-data`

## 四、P2 数据（锦上添花，提升体验）

### 4.1 岗位薪资/福利估算

- 当前岗位质量评分中的 `quality` 字段是硬编码的，应改为数据驱动

### 4.2 用户行为埋点
- 表单各步骤的流失率
- 结果页的付费转化率
- 建议接入 Google Analytics 或国内等效方案（百度统计/友盟）
