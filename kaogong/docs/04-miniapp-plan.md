# AI公考上岸助手 - 微信小程序方案

## 核心思路

**复用现有 Next.js 后端 API，小程序只做前端壳。**

```
现有架构：Web前端 → Next.js API → AI引擎
小程序架构：小程序前端 → 同一套 Next.js API → AI引擎
```

两端共享同一个后端，零重复开发引擎逻辑。

---

## 技术选型

| 项目 | 方案 | 理由 |
|------|------|------|
| 小程序框架 | **Taro 3** (React语法) | 现有组件逻辑可直接迁移复用 |
| 后端 | 现有 Next.js API Routes | 不新增服务，只需开放跨域 |
| 支付 | 微信小程序原生支付 | 小程序内必须用微信支付 |
| 登录 | wx.login + openId | 小程序标准授权流 |
