# AI公考上岸助手 - 产品需求文档（PRD）

## 一、产品概述

### 1.1 项目名称
AI公考上岸助手

### 1.2 产品定位
基于用户个人条件与历史公考数据，通过AI与规则引擎结合，提供"选岗决策建议"的工具型产品，帮助用户提高上岸概率，避免无效报考。

### 1.3 核心价值
- 帮助用户避开高风险岗位（减少试错成本）
- 提供结构化决策建议（替代人工咨询）
- 提升岗位选择效率（分钟级决策）

---

## 二、目标用户

### 2.1 用户画像
- 18–35岁公考考生
- 应届毕业生 / 往届考生
- 有一定焦虑但缺乏选岗经验

### 2.2 用户痛点
- 不知道自己能报哪些岗位
- 无法判断岗位竞争难度
- 容易选择"三不限"等高风险岗位
- 信息分散，决策成本高

---

## 三、产品目标

### 3.1 短期目标（0–1个月）
- 上线MVP版本
- 获取100个用户
- 实现首批付费转化

### 3.2 中期目标（1–3个月）
- 用户规模1000+
- 优化推荐算法
- 打磨用户体验

### 3.3 长期目标
- 构建考试决策平台（扩展教资/考研等）

---

## 四、产品结构

### 4.1 页面结构

#### 页面1：首页（信息输入页）
- 表单输入用户信息
- 提交生成报告

#### 页面2：结果页
- 展示基础分析（免费）
- 提供付费解锁完整报告

#### 页面3：支付页
- 用户完成支付
- 解锁完整内容

---

## 五、功能设计

### 5.1 用户输入模块

| 字段 | 类型 | 是否必填 |
|------|------|--------|
| 学历 | 下拉选择 | 是 |
| 是否全日制 | 单选 | 是 |
| 专业名称 | 文本 | 是 |
| 毕业时间 | 数字 | 是 |
| 是否应届 | 单选 | 是 |
| 政治面貌 | 单选 | 是 |
| 意向地区 | 文本 | 是 |
| 是否接受乡镇 | 单选 | 是 |
| 证书情况 | 单选 | 否 |
| 基层经历 | 单选 | 否 |
| 是否考过 | 单选 | 否 |
| 行测分数 | 数字 | 否 |
| 申论分数 | 数字 | 否 |
| 每日学习时间 | 数字 | 否 |

### 5.2 岗位筛选模块（规则引擎）
- if 年龄 > 岗位限制 → 剔除
- if 非应届报应届岗 → 剔除
- if 专业不匹配 → 剔除
- if 学历不满足 → 剔除

### 5.3 身份红利评分模块
- 应届生：+30
- 党员：+15
- 法考A证：+40
- 基层经历：+20
- 户籍匹配：+25
- 输出红利等级：T0 / T1 / 普通 / 三不限

### 5.4 竞争烈度评估模块
- 竞争指数 = 报录比 × 地域热度 × 专业池大小 × 岗位类型

### 5.5 岗位评分模块
- FinalScore = 0.4×匹配度 + 0.3×上岸概率 + 0.2×岗位质量 + 0.1×风险系数

### 5.6 AI分析模块
- 输出：红利等级、岗位推荐、竞争分析、避坑建议、备考策略

---

## 六、输出结果设计

### 6.1 免费内容
- 红利等级、岗位方向（不含具体岗位）、简要提示

### 6.2 付费内容
- Top5岗位推荐、每个岗位评分、竞争烈度分析、避坑建议、备考策略

---
