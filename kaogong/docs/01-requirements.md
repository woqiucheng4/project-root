# AI公考上岸助手 - 需求分析

## 一、核心需求梳理

### 1.1 产品本质
这是一个 **决策辅助工具**，核心逻辑是：
- 输入：用户个人条件（学历、专业、政治面貌等）
- 处理：规则引擎筛选 + AI 分析
- 输出：岗位推荐报告（免费基础版 + 付费完整版）

### 1.2 三大核心引擎

| 引擎 | 功能 | 输入 | 输出 |
|------|------|------|------|
| 规则引擎 | 岗位筛选与匹配 | 用户条件 + 岗位库 | 符合条件的岗位列表 |
| 评分引擎 | 多维度打分排序 | 筛选后岗位 + 用户画像 | 岗位排名与评分 |
| AI引擎 | 深度分析与建议 | 用户信息 + 评分结果 | 自然语言分析报告 |

### 1.3 业务流程

```
用户填写信息 → 规则引擎筛选岗位 → 身份红利评分 → 竞争烈度评估 → 岗位综合评分 → AI生成报告 → 展示结果(免费/付费)
```

## 二、功能模块分析

### 2.1 用户输入模块
**必填字段（8个）**：学历、是否全日制、专业名称、毕业时间、是否应届、政治面貌、意向地区、是否接受乡镇

**选填字段（6个）**：证书情况、基层经历、是否考过、行测分数、申论分数、每日学习时间

> 设计要点：表单需要分步引导，降低用户填写压力

### 2.2 岗位筛选模块（规则引擎）
硬性条件过滤，不符合直接剔除：
- 年龄超限 → 剔除
- 非应届报应届岗 → 剔除
- 专业不匹配 → 剔除
- 学历不满足 → 剔除

### 2.3 身份红利评分模块
加分项及分值：
- 应届生：+30
- 党员：+15
- 法考A证：+40
- 基层经历：+20
- 户籍匹配：+25

输出红利等级：T0 / T1 / 普通 / 三不限

### 2.4 竞争烈度评估模块
```
竞争指数 = 报录比 × 地域热度 × 专业池大小 × 岗位类型
```

参数配置：
- 地域热度：一线1.5 / 二线1.3 / 普通1.0 / 偏远0.7
- 专业池：法学+A证0.5 / 计算机0.8 / 汉语言1.5 / 三不限2.0
- 岗位类型：两办/纪委1.3 / 普通1.0 / 乡镇0.7

### 2.5 岗位综合评分模块
```
FinalScore = 0.4×匹配度 + 0.3×上岸概率 + 0.2×岗位质量 + 0.1×风险系数
```

### 2.6 AI分析报告模块
输出内容结构：红利等级 → 岗位推荐 → 竞争分析 → 避坑建议 → 备考策略

### 2.7 付费体系
- 免费：红利等级 + 岗位方向 + 简要提示
- 付费（9.9元）：Top5岗位推荐 + 评分 + 竞争分析 + 避坑建议 + 备考策略

## 三、MVP 范围界定

### 纳入 MVP
- ✅ 用户信息表单
- ✅ 规则引擎（基于配置规则，无需真实岗位数据库）
- ✅ 身份红利评分
- ✅ 竞争烈度评估
- ✅ AI报告生成（DeepSeek API）
- ✅ 免费/付费分层展示
- ✅ 支付功能（可先用模拟）

### 延后处理
- ❌ 真实岗位数据库（V2）
- ❌ 用户账号系统（V2）
- ❌ 小程序（V3）
- ❌ 1对1咨询、课程分销
