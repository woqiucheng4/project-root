# AI公考上岸助手 - 任务拆分

## 总览

MVP 开发共拆分为 **6个里程碑、22个任务**，预计总工时 5-7 天。

```
M1 项目初始化 ──→ M2 表单页面 ──→ M3 核心引擎 ──→ M4 AI集成 ──→ M5 结果页面 ──→ M6 支付与上线
   (0.5天)          (1天)          (1.5天)          (1天)         (1天)          (1天)
```

---

## M1 - 项目初始化（0.5天）

### T1.1 创建 Next.js 项目
- 使用 `npx create-next-app` 初始化项目
- 配置 App Router 模式
- 清理模板代码
- **产出**：可运行的空项目

### T1.2 设计系统搭建
- 创建 `globals.css` 设计令牌（颜色、字体、间距、阴影）
- 配置 Google Fonts（推荐 Inter + Noto Sans SC）
- 设计响应式断点
- 搭建深色/渐变主题风格
- **产出**：设计系统 CSS 文件

### T1.3 通用组件开发
- Header 导航栏组件
- Footer 组件
- Loading / 骨架屏组件
- AnimatedCard 动效卡片组件
- **产出**：可复用的基础组件库

### T1.4 全局布局搭建
- 创建 `layout.js` 全局布局
- 配置页面元数据（SEO）
- 配置页面切换动画
- **产出**：完整的页面框架

---

## M2 - 用户输入表单（1天）

### T2.1 表单配置文件
- 创建 `config/formFields.js`
- 定义所有字段的类型、选项、验证规则
- 区分必填/选填字段
- **产出**：可驱动表单渲染的配置数据

### T2.2 表单组件开发
- `FormField.jsx` - 通用表单字段组件（下拉、单选、文本、数字）
- `StepIndicator.jsx` - 分步进度指示器
- `UserInfoForm.jsx` - 主表单组件
- 实现分步表单（3步：基本信息 → 报考意向 → 补充信息）
- **产出**：完整的分步表单

### T2.3 表单验证
- 创建 `utils/validators.js`
- 必填字段校验
- 数据格式校验（分数范围、年份等）
- 实时错误提示
- **产出**：表单验证逻辑

### T2.4 首页UI打磨
- 页面顶部 Hero 区域（产品介绍 + 信任背书）
- 表单区域精美样式
- 提交按钮动效
- 移动端适配
- **产出**：上线质量的首页

---

## M3 - 核心引擎开发（1.5天）

### T3.1 评分规则配置
- 创建 `config/scoringRules.js` - 身份红利评分规则
- 创建 `config/competition.js` - 竞争参数配置
- 创建 `config/regions.js` - 地域热度配置
- **产出**：集中管理的规则配置文件

### T3.2 规则引擎开发
- 创建 `services/ruleEngine.js`
- 实现岗位类型筛选逻辑（学历、应届、专业等硬条件过滤）
- MVP阶段基于岗位类型而非具体岗位进行筛选
- 单元测试
- **产出**：可用的规则引擎

### T3.3 身份红利评分
- 创建 `services/bonusScoring.js`
- 实现评分加权逻辑
- 实现红利等级判定（T0/T1/普通/三不限）
- 单元测试
- **产出**：红利评分模块

### T3.4 竞争烈度评估
- 创建 `services/competition.js`
- 实现竞争指数计算公式
- 地域热度、专业池、岗位类型系数计算
- 单元测试
- **产出**：竞争评估模块

### T3.5 岗位综合评分
- 创建 `services/jobScoring.js`
- 实现加权评分公式：`0.4×匹配度 + 0.3×上岸概率 + 0.2×岗位质量 + 0.1×风险系数`
- 排序输出 Top5
- 单元测试
- **产出**：综合评分模块

### T3.6 分析API接口
- 创建 `app/api/analyze/route.js`
- 串联 规则引擎→红利评分→竞争评估→综合评分 流水线
- 输入验证
- 错误处理
- **产出**：`POST /api/analyze` 接口

---

## M4 - AI 集成（1天）

### T4.1 AI 分析模块
- 创建 `services/aiAnalyzer.js`
- 设计 System Prompt（公考选岗顾问角色）
- 构建 User Prompt 模板（注入用户信息+评分结果）
- 对接 DeepSeek API
- 解析 AI 返回结果为结构化数据
- **产出**：AI 分析服务

### T4.2 流式响应支持
- 分析接口支持 SSE（Server-Sent Events）
- 前端实现流式接收展示
- 错误降级处理（AI 超时/报错时返回基础报告）
- **产出**：流式 AI 响应能力

### T4.3 报告存储
- 使用内存 Map / 文件存储报告结果（MVP阶段）
- 生成唯一 reportId
- 区分免费/付费内容存储
- 创建 `GET /api/report` 查询接口
- **产出**：报告存取能力

---

## M5 - 结果页面（1天）

### T5.1 免费报告组件
- `BonusLevel.jsx` - 红利等级卡片（视觉化展示T0/T1等级）
- 岗位方向概述区域
- 简要提示区域
- **产出**：免费报告 UI

### T5.2 付费报告组件
- `JobRecommend.jsx` - Top5 岗位推荐卡片
- `CompetitionChart.jsx` - 竞争烈度可视化
- `AvoidPitfalls.jsx` - 避坑建议列表
- `StudyPlan.jsx` - 备考策略时间线
- **产出**：付费报告 UI

### T5.3 付费墙组件
- `PayWall.jsx` - 内容模糊遮罩 + 解锁引导
- `PriceCard.jsx` - 价格展示卡片
- 付费前后切换动画
- **产出**：付费引导 UI

### T5.4 结果页整合
- 创建 `result/page.js`
- 整合免费+付费内容
- AI 分析加载状态（流式打字效果）
- 分享功能（生成海报/链接）
- **产出**：完整的结果页

---

## M6 - 支付与上线（1天）

### T6.1 支付流程
- 创建 `payment/page.js`
- 创建 `app/api/payment/route.js`
- MVP阶段可先实现模拟支付（点击即解锁）

- 配置环境变量（DeepSeek API Key 等）

- 域名绑定
- **产出**：线上可访问的产品

## 技术风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| DeepSeek API 响应慢 | 用户等待时间长 | 流式返回 + 骨架屏 + 超时降级 |
| AI 输出格式不稳定 | 前端解析异常 | JSON Schema 约束 + 兜底解析 |
| MVP 无真实岗位数据 | 推荐不够精准 | 基于岗位类型泛化推荐 + 说明 |
| 移动端适配 | 用户体验差 | 移动端优先设计，分步表单 |
